## 2. ロール別：暫定モデル割り当て案

上記の比較結果を踏まえた、**現時点での暫定方針**です。

> この割り当ては「現時点の仮説」です。  
> 実際の挙動やコストを見ながら、`config/models.yaml` と合わせて継続的に見直します。

### カウンセラー（Counselor）

- 候補：
  - Anthropic Claude 系（Instant / Sonnet など）
  - OpenAI GPT-4 系を一部併用
- 理由：
  - 共感的な対話、柔らかい文体、安全性の高さを重視。

### 教師（Teacher）

- 候補：
  - OpenAI GPT-4 系を基本
  - サブ候補として Gemini / Claude を一部タスクで併用
- 理由：
  - 厳密な説明・推論・専門知識が必要なため、高精度モデルを優先。

### ファシリテーター（Facilitator）

- 候補：
  - Anthropic Claude 系（長文コンテキスト・要約）
  - 講義動画や録画の要約には Gemini 系も検討
- 理由：
  - 長時間ディスカッションや多数ログの要約・整理に強み。

### 実務作業・下位タスク（Trainee など）

- 候補：
  - OpenAI GPT-3.5 系
  - Claude Instant
  - Mistral 系 OSS（将来的な自前ホストも視野）
- 理由：
  - コストを抑えつつ、大量の軽めタスクやドラフト生成を行うため。

## 3. 今後の運用ポリシー（案）

- LLMの料金・性能は頻繁に変わるため：
  - 半年〜1年単位で本ドキュメントを見直す。
  - 実プロジェクトでのログを元に、
    - 「どのロールにどのモデルがハマったか」
    - 「どこでハルシネーションや事故が起きたか」
    を記録し、モデル割り当てを更新していく。
- 実装上は：
  - 具体的なモデル名・プロバイダは `config/models.yaml` に集約し、
  - 本ドキュメントは「なぜ今その設定なのか？」を説明する背景資料として扱う。
